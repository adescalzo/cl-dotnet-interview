# Status — 2026-04-20

A snapshot of where the repo is, not an ADR. For the decision record
see `docs/adr/`.

## Landed

`TodoLists` is the first Clean-Architecture / DDD slice and is closed
end-to-end across its CRUD surface.

- Aggregate: `TodoList` now has a private parameterless ctor, public
  ctor, private setters, and behavior methods (`Update`, `AddItem`,
  `RemoveItem`, `Synchronized`). `TodoItem` is still anemic — next.
- CQRS via Wolverine (ADR-0008). Shipped use cases: `CreateTodoList`,
  `UpdateTodoList`, `DeleteTodoList`, `GetTodoList`, `GetTodoLists`.
- Controllers split by side: `TodoListsQueryController` and
  `TodoListsCommandController` under `Controllers/TodoLists/{Queries,Commands}/`.
- `IRepositoryCommand<TodoList>` is backed by the existing
  `RepositoryCommand<TEntity>` base class. There is no concrete
  `IRepositoryQuery<TodoList>` in production yet — the query handlers
  resolve the interface at runtime; handler tests use a test-scoped
  stand-in in `TodoApi.Tests/TestSupport/`.
- Validation runs as a custom Wolverine middleware
  (`TodoApi.Infrastructure.Mediator.ValidationMiddleware`) that
  short-circuits the chain with `Result.Failure(ErrorResult.Validation(...))`.
  No exceptions for expected failures (ADR-0006). Validators live next
  to the command/query (`<UseCase>CommandValidator.cs`).
- HTTP error rendering: `ResultExtensions.ToProblemDetails` emits RFC
  7807 `ProblemDetails` / `ValidationProblemDetails` with `code` and
  `traceId` extensions (ADR-0007).
- Logging on Serilog (ADR-0009).
- Testing conventions shipped (ADR-0011): `AsyncLifetimeBase` over EF
  InMemory, `IBuilder<T>`, `TodoListBuilder` / `TodoItemBuilder`,
  FluentAssertions v6, NSubstitute. Handler + validator tests cover
  every TodoLists use case.
- `TodoItem.TodoListId` bumped to `Guid` so the EF model validates
  against the Guid-keyed `TodoList`; `TodoItemsController` updated to
  match (still starter-style, see below).

## Next

- `TodoItem` aggregate refactor: move it under the `TodoList` aggregate
  as a child entity with no separate repository; rework
  `TodoItemsController` into nested routes; mirror the TodoLists
  command/query/validator/test shape.

## Deferred

- Sync with the external system — not started, scope-gated per the
  "Current focus" note in `CLAUDE.md`.
- Replacing the legacy `TodoContext` / `Migrations/` / `Dtos/` starter
  layout with a module-local `Infrastructure/Persistence/` layout.
- A concrete production `IRepositoryQuery<TodoList>` implementation.
- Integration tests at the HTTP level — externally owned by
  `crunchloop/interview-tests`.
